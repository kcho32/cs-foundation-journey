# Stack, Queue
Stack과 Queue 여러 데이터를 순서대로 보관하는 구조

차이: Stack은 LIFO, Queue는 FIFO
LIFO: 최신 데이터가 더 빠르게 나가는 순서를 가짐 -> 입구와 출구가 하나 (접시 쌓기)
FIFO: 더 빨리 들어온 데이터가 더 빨리 나감 -> 양 측으로 입구와 출구가 나뉨 (줄 서기)
예시는 물리적인 예시가 아니라 허용된 연산의 규칙을 이미지화 한것

## Stack 
### 연산
push -> 데이터를 넣는다
pop -> 가장 최근에 넣은 데이터를 꺼낸다 
peek/top -> 가장 최근 데이터를 확인한다
isEmpty -> 비어 있는지 확인

### 사용처
보통 코드에서 함수 호출할 때 Stack 로직으로 흘러감 
ex: 함수 A에서 B호출 -> B에서 C호출 -> C호출 후 B러 나머지 로직 수행 -> B 로직 수행 후 A로 돌아가서 나머지 로직 수행

뒤로 가기/ Undo
브라우저 뒤로가기나 문서 편집기의 Undo도 가장 최근 것을 꺼내서 없던 일로한다는 것처럼 Stack

깊이 우선 탐색(DFS) - 최근 본 노드부터 처리

### 시간 복잡도
보통 Stack은 배열을 가지고 구현하기 때문에 대부분 O(1)이나
push인 경우에만 amortized O(1)의 복잡도를 가지고 있음

javascript이 shift같은 방식은 Queue 구현으로는 비효율적

## Queue
### 연산
enqueue 데이터를 뒤에 넣는다 (입구)
dequeue 데이터를 앞에서 꺼낸다 (출구)
front/peak 가장 오래된 데이터를 조회한다 (나갈 데이터)
isEmpty 비어 있는지 확인

### 사용처
먼저 온 일을 먼저 처리하는 상황

대기열: 들어온 순서대로 처리
너비 우선 탐색(BFS) - 먼저 본 노드부터 처리

### 시간 복잡도
보통 amortized O(1)으로 볼 수 있지만 만약 구현 방식이 데이터를 꺼내고 나머지 데이터를 앞으로 당기는 방식이면 O(n)으로 올라간다

## Stack과 Queue는 저장방식이 아닌 접근(사용) 규칙
메모리에 어떻게 저장되냐는 것이 아니라 데이터를 어떻게 넣고 빼냐는 규칙

* Stack과 Queue는 Array 기반으로 만들 수도, LinkedList 기반으로도 만들 수 있음

### Circular Queue 
효율적으로 Queue를 구현한 방식

일반 배열로 된 Queeue는 데이터를 빼면 공간이 비게 되는데 이 공간을 뒤의 데이터를 옮기는 방식으로 구현하면 남은 모든 데이터를 옮겨야하므로 O(n)의 복잡도를 가지게 됨

그래서 데이터를 움직이지 않고 큐를 사용하려면 인덱스를 사용해서 마지막 공간이 찼으면 새로 배열을 크게 만드는 대신 다시 인덱스를 0부터 시작하는 방식으로 구현하는 방식이 있다. 물론 여기서도 배열 안에 크기 이상의 데이터를 넣을 수는 없다.

실무에서는 꽉 찬 경우 내부 배열을 더 확장하는 dynamic circular buffer를 사용하기도 한다.

## 실무적 관점
### Queue
요청이 많은 작업이 있을 때, 이를 한번에 처리하려면 많은 성능이 필요한데 Queue를 이용하면 Queue에 요청이 들어온 순서로 넣으면
워커가 순서대로 하나씩 꺼내서 처리할 수 있다.

장점으로는 요청을 바로 처리할 필요가 없다는 점과, 트래픽이 몰려도 Queue에 쌓이는 것이기 때문에 버퍼 역할을 해준다.
또한 실패한 작업을 다시 큐에 넣어서 재시도할 수 있고 많이 쌓이면 처리량을 늘리기 위해 워커 수를 조절할 있다.

RabbitMQ, SQS는 대표적인 메시지 큐 시스템이고,
Kafka는 큐처럼 사용할 수 있지만 더 정확히는 로그 기반 이벤트 스트리밍 플랫폼이다.

## Stack Overflow
실제로 Stack에 함수 호출 정보가 너무 많이 쌓여서 스택 메모리 한계를 초과한 상황을 의미하며 실제 컴퓨터 과학의 개념 (무한 재귀함수)

## Stack VS Heap
### 자료구조의 Stack
데이터를 LIFO 방식으로 넣고 꺼내는 추상 자료구조이다.

메모리 Stack:
함수 호출마다 stack frame이 쌓이는 공간.
함수가 끝나면 해당 stack frame이 자동으로 제거된다.
지역 변수, 매개변수, 반환 위치, 참조값 등이 저장된다.

Heap:
동적으로 생성된 객체나 배열 같은 데이터가 저장되는 공간.
함수 호출이 끝났다고 바로 사라지는 게 아니라,
더 이상 참조되지 않을 때 GC나 수동 해제에 의해 정리된다.

### 메모리 Stack
함수 호출 정보, 반환 주소, 지역 변수 등이 저장되는 메모리 영역이다.
함수가 호출되면 stack frame이 쌓이고, 함수가 종료되면 제거된다.

### Heap
동적으로 생성된 객체나 데이터가 저장되는 메모리 영역이다.
Stack과 Heap은 서로 다른 메모리 영역이며, Stack이 Heap의 데이터를 저장/불러내는 방식은 아니다.
