# 공간 지역성 (Spatial Locality)
어떤 메모리 위치의 데이터를 사용하면, 가까운 메모리 위치의 데이터도 곧 사용할 가능성이 높다는 성질.
Array는 데이터가 연속적으로 저장되므로 공간 지역성이 좋다.

# 캐시 효율
CPU는 메모리에서 데이터를 하나씩만 가져오는 것이 아니라 cache line 단위로 주변 데이터까지 함께 가져온다.
Array처럼 관련 데이터가 연속된 메모리에 모여 있으면, 한 번 가져온 cache line 안에 다음에 사용할 데이터가 포함될 가능성이 높아진다.
그래서 순회 성능이 좋다.

효율이 나쁜 것은 데이터가 연속된 메모리에 있지 않는 경우를 말한다

# Pointer Chasing
LinkedList처럼 각 노드가 다음 노드의 주소를 가지고 있을 때, 다음 데이터를 읽기 위해 포인터를 따라 메모리 여기저기로 이동하는 과정.
노드들이 메모리에 흩어져 있으면 cache miss가 자주 발생할 수 있다.

#Big O
Big-O는 연산 횟수의 증가 추세를 설명하지만, 실제 성능은 메모리 접근 패턴, 캐시 효율, 객체 오버헤드, 언어 런타임 구현에 따라 달라질 수 있다.
따라서 Array와 LinkedList의 순회는 둘 다 O(n)이지만, 실제로는 Array가 캐시 친화적이어서 더 빠른 경우가 많다.
